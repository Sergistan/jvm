ClassLoader'ы:
JAVA подгружает классы JvmComprehension, Object, Integer, System:
1) сначала ищет в кэше Application ClassLoader -> затем ищет в кэше Platform ClassLoader -> затем ищет в кэше
Bootstrap ClassLoader -> класс нигде не найден 
2) в обратном порядке начинает искать классы: 
Bootstrap ClassLoader находит классы Object, Integer, System 
в Platform ClassLoade не находится ничего, т.к нет классов из java.util
в Application ClassLoader находит класс JvmComprehension

Память:
В Metaspace помещаются метаданные классов: JvmComprehension, Object, Integer, System.
1 строчка: в стеке создается фрейм main, в котором выделяется память и инициализируется 
примитивная переменная int i = 1; 
2 строчка: в куче создается объект new Object(); а во фрейме main выделяется память 
и инициализируется ссылочная переменная Object o (выше int i)
3 строчка: в куче создается объект Integer со значением 2, а во фрейме main выделяется память 
и инцилизируется ссылочная переменная Integer ii (выше int i и Object o)
4 строчка: вызывается метод printAll, следовательно создается фрейм printAll (выше фрейма main),
во фрейме printAll инициализируются ссылочные переменные Object o и Integer ii, ссылающееся в куче
на объекты new Object() и объект Integer со значением 2 соответственно, также
во фрейме printAll инициализируется примитивная переменная int i=1;
5 строчка: в куче создается объект Integer со значением 700 а во фрейме printAll выделяется память 
и инициализируется ссылочная переменная Integer uselessVar
6 строчка: вызывается статический метод System.out.println, следовательно создается фрейм println 
(выше фрейма main и printAll),
во фрейме out.println инициализируются ссылочные переменные Object o и Integer ii, ссылающееся в куче
на объекты new Object() и объект Integer со значением 2 соответственно, также
во фрейме printAll инициализируется примитивная переменная int i=1, затем происходит выполнение метода:
на экран выводится название объекта Object o с конкатенацией к имени значения переменных int i и Integer ii.
Фреймы println и printAll завершаются.
7 строчка: возвращаемся во фрейм main, вызывается статический метод System.out.println, 
следовательно создается фрейм println и в куче создается объект String со значением "finished".
На экран выводится значение "finished". Фреймы println и main завершаются.

Сборщик мусора:
Переменная int i = 1 удаляется из стэка (не сборщиком мусора) по завершению выполнения фрейма main.
Объект Integer uselessVar = 700 удаляется сборщиком мусора из кучи по завершению выполнения фрейма printAll.
Объекты Object o = new Object() И Integer ii = 2 удаляются из кучи по завершению выполнения фрейма main.
